# 内容工具与工作流引擎方案 V2.0

## 一、问题重新定义

### 1.1 核心矛盾

之前的方案将"工具"和"业务流程"混为一谈，导致架构设计不合理：

**工具（Tool）** vs **工作流（Workflow）**

| 维度 | 原子工具 | 业务工作流 |
|------|---------|-----------|
| **职责** | 单一功能执行 | 完成端到端业务目标 |
| **输入输出** | 明确的输入输出 | 涉及多步骤、多数据源 |
| **数据持久化** | 不涉及 | 需要保存到数据库 |
| **业务逻辑** | 无 | 包含业务规则和流程 |
| **示例** | AI生成文本、翻译、图片搜索 | 文章生成并发布、批量翻译文章 |

### 1.2 实际需求场景

#### 场景1: 文章编辑器中使用工具
- **用户**: 编辑文章时选中一段文字
- **需求**: 优化这段文字（不保存，只是替换编辑器内容）
- **本质**: 使用**原子工具**辅助编辑

#### 场景2: 批量生成文章
- **用户**: 从标题池批量生成文章
- **需求**: 
  1. 从标题池获取标题
  2. 调用AI生成内容
  3. 处理生成结果
  4. 保存为文章（草稿或发布）
  5. 可能涉及图片搜索、SEO优化等
- **本质**: 执行**业务工作流**

#### 场景3: 翻译已有文章
- **用户**: 将一篇文章翻译成另一种语言
- **需求**:
  1. 读取源文章
  2. 翻译标题、内容、摘要
  3. 保存为新文章
- **本质**: 执行**业务工作流**

#### 场景4: 生成游戏主页
- **用户**: 为某个游戏生成主页内容
- **需求**:
  1. 读取游戏信息
  2. 使用模板生成主页内容
  3. 保存/发布主页
- **本质**: 执行**业务工作流**

#### 场景5: 为多个游戏批量生成推广文章（新增）⭐
- **用户**: 在游戏管理页面选择多个游戏
- **需求**:
  1. 传入游戏ID列表（如：[101, 102, 103]）
  2. 循环处理每个游戏：
     - 从系统读取游戏详细信息（名称、类型、介绍等）
     - 使用AI生成游戏推广文章
     - 生成SEO关键词
     - 搜索相关配图
     - 保存并发布文章
     - 关联游戏和文章
  3. 返回生成结果统计
- **本质**: 执行**业务工作流**，支持系统内部数据输入

### 2.2 核心概念

#### 2.2.1 原子工具（Atomic Tool）

**定义**：独立的、可复用的功能单元，完成单一明确的任务

**特点**：
- 无状态（stateless）
- 单一职责
- 可组合
- 不涉及业务数据持久化

**数据结构**：
```java
public class AtomicTool {
    private Long id;
    private String toolCode;      // 工具唯一标识：ai_generator, translator, image_search
    private String toolName;      // 工具名称
    private String toolType;      // 工具类型：ai, api, builtin
    private String description;   // 描述
    
    // 工具配置
    private ToolConfig config;    // 包含API配置、提示词等
    
    // 输入输出定义
    private List<ToolParameter> inputs;   // 输入参数定义
    private List<ToolParameter> outputs;  // 输出参数定义
}

// 工具参数定义
public class ToolParameter {
    private String name;          // 参数名
    private String type;          // 类型：string, text, number, array, object
    private boolean required;     // 是否必需
    private Object defaultValue;  // 默认值
    private String description;   // 描述
}
```

**示例工具**：

1. **AI文本生成工具**
   - 输入：`prompt` (string), `max_tokens` (number), `temperature` (number)
   - 输出：`generated_text` (string)

2. **翻译工具**
   - 输入：`text` (string), `source_lang` (string), `target_lang` (string)
   - 输出：`translated_text` (string)

3. **图片搜索工具**
   - 输入：`keyword` (string), `count` (number)
   - 输出：`images` (array)

4. **关键词提取工具**
   - 输入：`title` (string), `content` (string)
   - 输出：`keywords` (string)

#### 2.2.2 工作流（Workflow）

**定义**：由多个步骤组成的业务流程，每个步骤可以是工具调用或业务操作

**特点**：
- 有状态（stateful）
- 包含业务逻辑
- 步骤间数据传递
- 涉及数据持久化

**数据结构**：
```java
public class Workflow {
    private Long id;
    private String workflowCode;     // 工作流唯一标识
    private String workflowName;     // 工作流名称
    private String description;      // 描述
    private Long siteId;             // 所属网站
    
    // 工作流定义（JSON）
    private WorkflowDefinition definition;
    
    // 配置
    private boolean enabled;         // 是否启用
    private String triggerType;      // 触发方式：manual, scheduled, event
}

// 工作流定义
public class WorkflowDefinition {
    private List<WorkflowStep> steps;          // 步骤列表
    private Map<String, Object> globalConfig;  // 全局配置
}

// 工作流步骤
public class WorkflowStep {
    private String stepId;           // 步骤ID
    private String stepName;         // 步骤名称
    private String stepType;         // 步骤类型：tool, condition, loop, save
    
    // 对于tool类型
    private String toolCode;         // 调用的工具代码
    private Map<String, Object> toolParams;  // 工具参数（可使用变量）
    
    // 对于condition类型
    private String condition;        // 条件表达式
    private String nextStepIfTrue;   // 条件为真时的下一步
    private String nextStepIfFalse;  // 条件为假时的下一步
    
    // 对于loop类型
    private String loopSource;       // 循环数据源
    private List<WorkflowStep> loopSteps;  // 循环内的步骤
    
    // 对于save类型
    private String entityType;       // 保存的实体类型：article, homepage
    private Map<String, String> fieldMapping;  // 字段映射
    
    // 输出映射（将步骤输出存储到工作流上下文）
    private Map<String, String> outputMapping;
    
    // 错误处理
    private String onError;          // 错误处理：continue, stop, retry
    private int retryCount;          // 重试次数
}
```

**工作流执行上下文**：
```java
public class WorkflowContext {
    private Long workflowId;
    private Long executionId;        // 本次执行的ID
    private Long siteId;
    
    // 上下文变量（步骤间共享数据）
    private Map<String, Object> variables;
    
    // 输入参数
    private Map<String, Object> inputs;
    
    // 当前步骤
    private String currentStepId;
    
    // 执行历史
    private List<StepExecutionRecord> executionHistory;
}
```

#### 2.4.2 工作流执行器实现

```java
@Service
public class WorkflowEngineServiceImpl implements IWorkflowEngineService {
    
    @Autowired
    private IWorkflowService workflowService;
    
    @Autowired
    private IAtomicToolService toolService;
    
    @Autowired
    private WorkflowExecutionLogService logService;
    
    @Override
    public WorkflowExecutionResult execute(String workflowCode, Map<String, Object> inputs) {
        // 1. 加载工作流定义
        Workflow workflow = workflowService.getByCode(workflowCode);
        if (workflow == null) {
            throw new WorkflowNotFoundException(workflowCode);
        }
        
        // 2. 创建执行上下文
        WorkflowContext context = new WorkflowContext();
        context.setWorkflowId(workflow.getId());
        context.setInputs(inputs);
        context.setVariables(new HashMap<>());
        
        // 3. 创建执行记录
        WorkflowExecution execution = logService.createExecution(workflow.getId(), inputs);
        context.setExecutionId(execution.getId());
        
        try {
            // 4. 执行步骤
            WorkflowDefinition definition = workflow.getDefinition();
            for (WorkflowStep step : definition.getSteps()) {
                executeStep(step, context);
            }
            
            // 5. 标记执行成功
            logService.markSuccess(execution.getId(), context.getVariables());
            
            return WorkflowExecutionResult.success(context.getVariables());
            
        } catch (Exception e) {
            // 6. 记录执行失败
            logService.markFailed(execution.getId(), e.getMessage());
            throw new WorkflowExecutionException("工作流执行失败", e);
        }
    }
    
    private void executeStep(WorkflowStep step, WorkflowContext context) {
        // 记录步骤开始
        logService.recordStepStart(context.getExecutionId(), step.getStepId());
        
        try {
            switch (step.getStepType()) {
                case "tool":
                    executeTool(step, context);
                    break;
                case "condition":
                    executeCondition(step, context);
                    break;
                case "loop":
                    executeLoop(step, context);
                    break;
                case "save":
                    executeSave(step, context);
                    break;
                case "workflow":
                    executeSubWorkflow(step, context);
                    break;
                default:
                    throw new UnsupportedStepTypeException(step.getStepType());
            }
            
            // 记录步骤成功
            logService.recordStepSuccess(context.getExecutionId(), step.getStepId());
            
        } catch (Exception e) {
            // 记录步骤失败
            logService.recordStepFailed(context.getExecutionId(), step.getStepId(), e.getMessage());
            
            // 根据错误处理策略决定是否继续
            if ("continue".equals(step.getOnError())) {
                return;
            } else if ("retry".equals(step.getOnError())) {
                // 重试逻辑
                for (int i = 0; i < step.getRetryCount(); i++) {
                    try {
                        executeStep(step, context);
                        return;
                    } catch (Exception retry) {
                        if (i == step.getRetryCount() - 1) {
                            throw retry;
                        }
                    }
                }
            } else {
                throw e;
            }
        }
    }
    
    private void executeTool(WorkflowStep step, WorkflowContext context) {
        // 1. 解析参数（替换变量）
        Map<String, Object> params = resolveParams(step.getToolParams(), context);
        
        // 2. 执行工具
        Map<String, Object> result = toolService.execute(step.getToolCode(), params);
        
        // 3. 保存输出到上下文
        if (step.getOutputMapping() != null) {
            for (Map.Entry<String, String> entry : step.getOutputMapping().entrySet()) {
                String outputKey = entry.getKey();
                String contextPath = entry.getValue();
                setContextValue(context, contextPath, result.get(outputKey));
            }
        }
    }
    
    private void executeLoop(WorkflowStep step, WorkflowContext context) {
        // 1. 获取循环数据源
        Object loopData = getContextValue(context, step.getLoopSource());
        
        if (!(loopData instanceof List)) {
            throw new WorkflowExecutionException("循环数据源必须是列表");
        }
        
        List<?> items = (List<?>) loopData;
        
        // 2. 遍历执行
        for (int i = 0; i < items.size(); i++) {
            Object item = items.get(i);
            
            // 设置循环变量
            context.getVariables().put("loop.index", i);
            context.getVariables().put("loop.item", item);
            
            // 执行循环内的步骤
            for (WorkflowStep loopStep : step.getLoopSteps()) {
                executeStep(loopStep, context);
            }
        }
        
        // 清除循环变量
        context.getVariables().remove("loop.index");
        context.getVariables().remove("loop.item");
    }
    
    private void executeSave(WorkflowStep step, WorkflowContext context) {
        // 1. 解析字段映射
        Map<String, Object> entityData = new HashMap<>();
        for (Map.Entry<String, String> entry : step.getFieldMapping().entrySet()) {
            String field = entry.getKey();
            String valuePath = entry.getValue();
            Object value = resolveValue(valuePath, context);
            entityData.put(field, value);
        }
        
        // 2. 根据实体类型保存
        switch (step.getEntityType()) {
            case "article":
                saveArticle(entityData, context);
                break;
            case "game_homepage":
                saveGameHomepage(entityData, context);
                break;
            default:
                throw new UnsupportedEntityTypeException(step.getEntityType());
        }
    }
    
    // 变量解析相关方法
    private Map<String, Object> resolveParams(Map<String, Object> params, WorkflowContext context) {
        Map<String, Object> resolved = new HashMap<>();
        for (Map.Entry<String, Object> entry : params.entrySet()) {
            resolved.put(entry.getKey(), resolveValue(entry.getValue(), context));
        }
        return resolved;
    }
    
    private Object resolveValue(Object value, WorkflowContext context) {
        if (value instanceof String) {
            String str = (String) value;
            // 支持变量替换：${variable.path}
            if (str.startsWith("${") && str.endsWith("}")) {
                String path = str.substring(2, str.length() - 1);
                return getContextValue(context, path);
            }
        }
        return value;
    }
    
    private Object getContextValue(WorkflowContext context, String path) {
        // 支持路径访问：input.title, article.content, loop.item.name
        String[] parts = path.split("\\.");
        Object current = null;
        
        if ("input".equals(parts[0])) {
            current = context.getInputs();
        } else {
            current = context.getVariables();
        }
        
        for (int i = 1; i < parts.length; i++) {
            if (current instanceof Map) {
                current = ((Map<?, ?>) current).get(parts[i]);
            } else if (current instanceof List) {
                try {
                    int index = Integer.parseInt(parts[i]);
                    current = ((List<?>) current).get(index);
                } catch (NumberFormatException e) {
                    throw new WorkflowExecutionException("无效的数组索引: " + parts[i]);
                }
            } else {
                // 使用反射访问对象属性
                current = getFieldValue(current, parts[i]);
            }
        }
        
        return current;
    }
}

### 3.4 工作流编辑方案

#### 3.4.1 三种编辑方式

**方式1: JSON配置编辑（初期推荐）**

优点：
- 开发成本低，无需开发可视化编辑器
- 灵活性高，可以精确控制每个细节
- 版本控制友好（Git diff清晰）
- 方便复制、模板化

缺点：
- 需要一定学习成本
- 容易出现格式错误
- 不够直观

适用场景：
- 系统初期快速迭代
- 技术人员配置工作流
- 需要精确控制的复杂工作流

**方式2: 表单配置编辑（中期推荐）**

优点：
- 用户友好，无需了解JSON
- 自动验证，减少错误
- 适合非技术人员使用
- 可以提供智能提示

缺点：
- 灵活性有限
- 复杂工作流配置繁琐

适用场景：
- 运营人员配置标准工作流
- 基于模板的工作流定制
- 常用工作流的快速配置

**方式3: 可视化流程编辑器（长期目标）**

优点：
- 最直观，所见即所得
- 图形化展示流程逻辑
- 支持拖拽操作
- 易于理解和维护

缺点：
- 开发成本高
- 复杂流程可能显示混乱

适用场景：
- 复杂业务流程配置
- 需要多人协作设计
- 展示给非技术决策者

## 四、使用场景对比

### 4.1 场景1: 编辑文章时优化一段文字

**使用方式**: 原子工具

```
用户操作：
1. 在编辑器中选中文字
2. 打开工具面板
3. 选择"文本优化"工具
4. 点击执行
5. 预览结果
6. 点击"替换"按钮

系统处理：
- 调用原子工具API
- 返回优化后的文字
- 前端替换编辑器内容
- 不涉及数据库保存
```

### 4.2 场景2: 批量生成100篇文章

**使用方式**: 工作流

```
用户操作：
1. 进入"批量生成"页面
2. 选择"批量文章生成"工作流
3. 配置参数：
   - 标题池
   - 目标网站
   - 分类
   - 是否发布
   - 生成数量
4. 点击"开始执行"
5. 查看执行进度

系统处理：
- 工作流引擎执行
- 循环处理每个标题
- 调用多个原子工具
- 保存文章到数据库
- 记录执行日志
```

### 4.3 场景3: 翻译文章到多种语言

**使用方式**: 工作流（可批量）

```
用户操作：

3. 选择目标语言（可多选）

系统处理：
- 对每种语言执行翻译工作流
- 读取源文章
- 调用翻译工具翻译各个字段
- 保存为新文章

## 五、优势总结

### 5.1 清晰的职责分离

- **原子工具**: 专注于单一功能，可复用
- **工作流**: 编排业务流程，完成端到端任务

### 5.2 灵活的组合能力

- 原子工具可以在多个工作流中复用
- 工作流可以嵌套调用
- 支持条件分支和循环

### 5.3 强大的扩展性

- 新增原子工具不影响现有工作流
- 新增工作流不需要改代码
- 支持自定义工作流

### 5.4 完善的监控和调试

- 执行记录详细
- 每个步骤都有日志
- 支持失败重试
- 可以查看中间结果

### 5.5 适应不同场景

- 编辑器辅助：使用原子工具
- 批量任务：使用工作流
- 自动化：定时触发工作流

## 七、总结

新方案的核心思想：

1. **原子工具 = 乐高积木**：小而美，可复用
2. **工作流 = 搭建模型**：组合积木完成复杂任务
3. **编辑器集成 = 手工工具**：辅助内容编辑
4. **批量任务 = 自动化生产线**：大规模内容生产

这样的设计既满足了编辑器中的辅助功能需求，又满足了批量自动化的需求，职责清晰，扩展性强。
