# 原子工具与工作流使用指南

## 一、架构设计

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                   原子工具层（Atomic Tools）                  │
│  单一职责的功能模块，不关心业务流程，只负责输入→处理→输出    │
├─────────────────────────────────────────────────────────────┤
│  AI生成    │ 翻译    │ 关键词提取 │ 图片搜索 │ 文本优化    │
│  (ai)     │ (ai)   │ (ai)      │ (api)   │ (ai)       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                  工作流引擎层（Workflow Engine）              │
│  编排多个原子工具，处理业务逻辑，与系统数据对接               │
├─────────────────────────────────────────────────────────────┤
│  文章生成工作流  │  批量关键词  │  游戏主页  │  文章翻译   │
│  (组合多工具)   │  生成工作流  │  生成工作流│  工作流     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                     业务场景层（Business）                    │
│  用户触发工作流执行、查看执行进度和结果、审核和发布内容       │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心概念

#### 原子工具（Atomic Tool）
- **定义**：单一职责的功能单元，如AI生成、翻译、图片搜索等
- **特点**：无状态、可复用、参数化配置
- **职责**：接收输入参数 → 执行处理逻辑 → 返回输出结果
- **配置位置**：AI配置 → 原子工具管理

#### 工作流（Workflow）
- **定义**：多个原子工具的编排组合，实现完整业务流程
- **特点**：支持顺序、循环、条件、数据保存等复杂逻辑
- **职责**：接收业务输入 → 调用工具链 → 处理系统数据 → 返回业务结果
- **配置位置**：内容管理 → 工作流管理

### 1.3 工作流步骤类型

| 步骤类型 | 说明 | 用途 |
|---------|------|------|
| `tool` | 调用原子工具 | 执行AI生成、翻译、图片搜索等功能 |
| `condition` | 条件判断 | 根据条件执行不同分支 |
| `loop` | 循环处理 | 批量处理多个数据项 |
| `save` | 数据操作 | 查询、插入、更新系统数据 |
| `workflow` | 嵌套工作流 | 调用其他工作流，实现复用 |

## 二、典型使用场景

### 场景1：批量生成SEO关键词

**业务需求**：
- 从标题池批次中获取所有标题
- 为每个标题生成5个SEO关键词
- 更新标题池记录，不涉及文章保存

**工作流配置**：

```json
{
  "workflowCode": "batch_generate_keywords",
  "workflowName": "批量生成SEO关键词",
  "description": "为标题池批次中的所有标题生成SEO关键词",
  "triggerType": "manual",
  "inputs": [
    {
      "name": "titlePoolBatchId",
      "type": "number",
      "label": "标题池批次ID",
      "required": true,
      "description": "要生成关键词的标题池批次ID"
    }
  ],
  "steps": [
    {
      "stepId": "step1",
      "stepName": "获取标题列表",
      "stepType": "save",
      "toolParams": {
        "action": "query",
        "table": "gb_title_pool",
        "condition": {
          "batch_id": "${input.titlePoolBatchId}",
          "del_flag": "0"
        },
        "select": ["id", "title", "language"]
      },
      "outputMapping": {
        "titles": "${step1.output.data}"
      }
    },
    {
      "stepId": "step2",
      "stepName": "循环生成关键词",
      "stepType": "loop",
      "loopConfig": {
        "items": "${step1.output.titles}",
        "itemVar": "title"
      },
      "steps": [
        {
          "stepId": "step2_1",
          "stepName": "调用AI生成关键词",
          "stepType": "tool",
          "toolCode": "ai_keyword_generator",
          "toolParams": {
            "title": "${title.title}",
            "language": "${title.language}",
            "count": 5
          }
        },
        {
          "stepId": "step2_2",
          "stepName": "更新标题池记录",
          "stepType": "save",
          "toolParams": {
            "action": "update",
            "table": "gb_title_pool",
            "condition": {
              "id": "${title.id}"
            },
            "data": {
              "seo_keywords": "${step2_1.output.keywords}",
              "update_time": "${now()}"
            }
          }
        }
      ]
    }
  ]
}
```

**页面集成代码**：

```vue
<!-- 标题池管理页面 -->
<template>
  <el-button 
    type="primary" 
    @click="handleBatchGenerateKeywords"
    :loading="generating"
  >
    批量生成关键词
  </el-button>
</template>

<script setup>
import { executeWorkflow, getExecutionStatus } from '@/api/gamebox/workflow'

const generating = ref(false)

async function handleBatchGenerateKeywords() {
  if (!selectedBatchId.value) {
    ElMessage.warning('请先选择一个批次')
    return
  }
  
  generating.value = true
  
  try {
    // 执行工作流
    const res = await executeWorkflow({
      workflowCode: 'batch_generate_keywords',
      mode: 'async',
      inputData: {
        titlePoolBatchId: selectedBatchId.value
      }
    })
    
    // 轮询执行状态
    pollExecutionStatus(res.data.executionId)
  } catch (error) {
    ElMessage.error('执行失败: ' + error.message)
    generating.value = false
  }
}

function pollExecutionStatus(executionId) {
  const timer = setInterval(async () => {
    const statusRes = await getExecutionStatus({ executionId })
    
    if (statusRes.data.status === 'success') {
      clearInterval(timer)
      generating.value = false
      ElMessage.success('关键词生成完成')
      getList() // 刷新列表
    } else if (statusRes.data.status === 'failed') {
      clearInterval(timer)
      generating.value = false
      ElMessage.error('生成失败: ' + statusRes.data.error)
    }
  }, 2000)
}
</script>
```

## 三、工作流配置详解

### 3.1 基本结构

```json
{
  "workflowCode": "工作流唯一编码",
  "workflowName": "工作流显示名称",
  "description": "工作流描述",
  "triggerType": "manual|scheduled|event",
  "inputs": [],     // 输入参数定义
  "steps": []       // 执行步骤定义
}
```

### 3.2 输入参数（inputs）

```json
{
  "name": "参数名称（变量名）",
  "type": "string|number|boolean|array|object",
  "label": "显示标签",
  "required": true|false,
  "default": "默认值",
  "description": "参数说明",
  "options": ["选项1", "选项2"]  // select类型的选项
}
```

### 3.3 步骤配置（steps）

#### tool 步骤 - 调用原子工具

```json
{
  "stepId": "step1",
  "stepName": "步骤名称",
  "stepType": "tool",
  "toolCode": "原子工具编码",
  "toolParams": {
    "参数名": "参数值或变量引用 ${variable}"

  }
}
```

```json

  "ifFalse": ["step5"]             // 条件为假执行的步骤ID

```

#### loop 步骤 - 循环处理

```json

  },

    // 循环体内的步骤

      "stepId": "loop_step1",
      "stepName": "处理单项",
      "stepType": "tool",
      "toolParams": {
        "data": "${item}"
      }
    }
  ]
}
```

#### workflow 步骤 - 嵌套工作流

```json

  "workflowCode": "子工作流编码",

  }
}
```

### 3.4 变量引用

工作流中支持以下变量引用：

| 变量类型 | 格式 | 示例 | 说明 |
|---------|------|------|------|
| 输入参数 | `${input.参数名}` | `${input.titleId}` | 工作流输入 |
| 步骤输出 | `${stepId.output.路径}` | `${step1.output.data.title}` | 步骤执行结果 |
| 循环变量 | `${itemVar.字段}` | `${item.id}` | 循环当前项 |
| 系统函数 | `${now()}` | `${now()}` | 当前时间 |
| 系统函数 | `${uuid()}` | `${uuid()}` | 生成UUID |

### 3.5 输出映射（outputMapping）

将步骤输出保存为工作流级别的变量：

```json
{
  "outputMapping": {
    "customVarName": "${step1.output.data.field}",
    "anotherVar": "${step2.output.result}"
  }
}
```

后续步骤可以引用：`${customVarName}`

## 六、执行模式

### 6.1 同步执行（sync）

- 适用场景：快速处理、需要立即获取结果
- 特点：阻塞等待，直到工作流完成
- 示例：文本优化、单个文章生成

```javascript
const res = await executeWorkflow({
  workflowCode: 'text_optimizer',
  mode: 'sync',
  inputData: { selectedText: '...' }
})

// 立即获取结果
console.log(res.data.outputData)
```

### 6.2 异步执行（async）

- 适用场景：耗时操作、批量处理
- 特点：立即返回执行ID，通过轮询获取状态
- 示例：批量生成关键词、文章翻译

```javascript
const res = await executeWorkflow({
  workflowCode: 'batch_generator',
  mode: 'async',
  inputData: { batchId: 123 }
})

// 返回执行ID
const executionId = res.data.executionId

// 轮询状态
const timer = setInterval(async () => {
  const status = await getExecutionStatus({ executionId })
  
  if (status.data.status === 'success') {
    clearInterval(timer)
    console.log('执行完成', status.data.outputData)
  }
}, 2000)
```

### 6.3 批量执行（batch）

- 适用场景：大量数据处理
- 特点：并行执行多个实例
- 示例：为100个标题批量生成文章

```javascript
const items = titleList.value // 100个标题

const res = await executeWorkflow({
  workflowCode: 'article_generator',
  mode: 'batch',
  inputData: items.map(item => ({
    titleId: item.id,
    siteId: currentSite.value
  }))
})

// 返回批次ID和执行ID列表
console.log(res.data.batchId, res.data.executionIds)
```

## 七、最佳实践

### 7.1 原子工具设计原则

1. **单一职责**：每个工具只做一件事
2. **无状态**：不依赖外部状态，只通过参数传递数据
3. **可测试**：提供测试接口，独立验证功能
4. **参数化**：通过参数控制行为，而不是硬编码
5. **标准输出**：统一的输出格式，便于工作流组合

### 7.2 工作流设计原则

1. **清晰的步骤划分**：每个步骤职责明确
2. **合理的错误处理**：添加条件判断和错误处理步骤
3. **适当的抽象层次**：复杂流程拆分为子工作流
4. **性能优化**：耗时操作使用异步执行
5. **可维护性**：添加详细的步骤说明和注释

### 7.3 常见问题处理

#### 问题1：工作流执行失败

**排查步骤**：
1. 检查输入参数是否正确
2. 查看步骤执行日志，定位失败步骤
3. 验证原子工具是否正常工作
4. 检查数据库连接和表权限

#### 问题2：变量引用错误

**解决方案**：
- 使用正确的变量路径：`${stepId.output.field}`
- 确保前置步骤已执行完成
- 检查JSON路径是否正确

#### 问题3：循环处理超时

**解决方案**：
- 减少单批次处理数量
- 使用批量执行模式
- 增加超时时间配置
- 添加断点续传机制
