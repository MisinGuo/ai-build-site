# 工作流工具编排系统使用指南

## 概述

本系统提供了可视化的工作流编排功能，允许用户通过拖拽和配置的方式，将多个原子工具组织成完整的工作流，实现复杂的自动化任务。

## 核心功能

### 1. 可视化工作流编辑器

- **工具面板**：展示所有可用的原子工具
  - 按分类组织工具
  - 支持搜索和过滤
  - 显示工具的输入输出参数

- **画布区域**：拖拽式工作流设计
  - 拖拽工具到画布添加步骤
  - 可视化展示步骤连接关系
  - 显示输入输出映射预览

- **属性面板**：配置步骤详细信息
  - 配置步骤名称和描述
  - 设置输入参数映射
  - 配置高级选项（超时、错误处理等）

### 2. 参数映射系统

支持三种数据源类型：

#### 常量值 (Constant)
直接输入固定的值，不依赖其他步骤或输入

```json
{
  "source": "constant",
  "value": "固定值"
}
```

#### 工作流输入 (Input)
从工作流的输入参数中获取值

```json
{
  "source": "input",
  "value": "input.siteId"  // 访问工作流输入的 siteId 参数
}
```

#### 前置步骤输出 (Step Output)
从之前步骤的输出中获取值

```json
{
  "source": "step_1",
  "value": "step_1.output.title"  // 访问步骤1输出的 title 字段
}
```

### 3. 工作流定义结构

```json
{
  "inputs": [
    {
      "name": "siteId",
      "type": "number",
      "label": "目标站点",
      "required": true,
      "default": null,
      "description": "文章发布的目标站点ID"
    }
  ],
  "steps": [
    {
      "stepId": "step_1",
      "stepName": "获取标题",
      "toolCode": "get_title_from_pool",
      "description": "从标题池获取一个未使用的标题",
      "inputMapping": {
        "siteId": {
          "source": "input",
          "value": "input.siteId"
        }
      },
      "enabled": true,
      "continueOnError": false,
      "timeout": 300
    },
    {
      "stepId": "step_2",
      "stepName": "AI生成文章",
      "toolCode": "ai_article_generator",
      "inputMapping": {
        "title": {
          "source": "step_1",
          "value": "step_1.output.title"
        }
      },
      "enabled": true,
      "continueOnError": false
    }
  ],
  "outputs": [
    {
      "name": "articleId",
      "source": "step_3.output.articleId",
      "description": "发布的文章ID"
    }
  ]
}
```

## 使用流程

### 步骤1：创建工作流

1. 在工作流列表页点击【新增】按钮
2. 填写工作流基本信息：
   - 工作流编码：唯一标识符
   - 工作流名称：显示名称
   - 描述：工作流的用途说明
   - 触发类型：手动/定时/事件

### 步骤2：配置工作流输入

1. 点击画布顶部的"工作流输入"节点
2. 点击【编辑】按钮
3. 添加输入参数：
   - 参数名：英文标识符（如 siteId）
   - 显示名称：用户友好的名称
   - 类型：string/number/boolean等
   - 是否必填
   - 默认值

### 步骤3：添加工作流步骤

#### 方法一：拖拽添加
1. 从左侧工具面板找到需要的工具
2. 拖拽到画布中间区域
3. 自动创建新步骤

#### 方法二：手动添加
1. 点击画布顶部的【添加步骤】按钮
2. 在弹出对话框中选择工具
3. 配置步骤信息

### 步骤4：配置步骤参数映射

选中步骤后，在右侧属性面板配置：

1. **选择数据源类型**：
   - 常量值：直接输入
   - 工作流输入：从下拉列表选择
   - 步骤输出：选择前置步骤的输出字段

2. **配置映射值**：
   - 常量：直接输入值
   - 变量：从可用变量列表选择

3. **验证映射**：
   - 系统会自动验证必填参数
   - 检查类型匹配
   - 提示映射错误

### 步骤5：配置工作流输出

1. 点击画布底部的"工作流输出"节点
2. 点击【编辑】按钮
3. 添加输出参数：
   - 输出名：输出参数名称
   - 数据源：选择某个步骤的输出字段
   - 描述：输出说明

### 步骤6：验证和测试

#### 验证工作流
点击【验证】按钮，系统会检查：
- 工作流基本信息完整性
- 所有步骤配置正确性
- 参数映射有效性
- 必填参数完整性

#### 模拟执行
点击【模拟执行】按钮：
- 输入测试数据
- 模拟执行流程（不保存实际数据）
- 查看每个步骤的输入输出
- 验证数据流转正确性

### 步骤7：保存和发布

1. 确认所有配置正确
2. 点击【保存】按钮
3. 启用工作流
4. 开始使用

## 示例：标题生成文章工作流

### 业务场景
从标题池获取标题 → AI生成文章 → 发布文章到网站

### 工作流配置

```javascript
{
  "workflowName": "标题生成文章",
  "workflowCode": "title_to_article",
  "description": "从标题池获取标题，使用AI生成文章内容，并发布到指定网站",
  
  "inputs": [
    {
      "name": "siteId",
      "type": "number",
      "label": "目标站点",
      "required": true
    },
    {
      "name": "categoryId",
      "type": "number",
      "label": "文章分类",
      "required": false
    }
  ],
  
  "steps": [
    {
      "stepId": "step_1",
      "stepName": "获取标题",
      "toolCode": "get_title_from_pool",
      "inputMapping": {
        "siteId": {"source": "input", "value": "input.siteId"},
        "categoryId": {"source": "input", "value": "input.categoryId"}
      }
    },
    {
      "stepId": "step_2", 
      "stepName": "AI生成文章",
      "toolCode": "ai_article_generator",
      "inputMapping": {
        "title": {"source": "step_1", "value": "step_1.output.title"},
        "keywords": {"source": "step_1", "value": "step_1.output.keywords"}
      }
    },
    {
      "stepId": "step_3",
      "stepName": "发布文章",
      "toolCode": "publish_article",
      "inputMapping": {
        "siteId": {"source": "input", "value": "input.siteId"},
        "categoryId": {"source": "input", "value": "input.categoryId"},
        "title": {"source": "step_1", "value": "step_1.output.title"},
        "content": {"source": "step_2", "value": "step_2.output.generatedText"}
      }
    }
  ],
  
  "outputs": [
    {
      "name": "articleId",
      "source": "step_3.output.articleId",
      "description": "发布的文章ID"
    },
    {
      "name": "articleUrl",
      "source": "step_3.output.articleUrl",
      "description": "文章访问URL"
    }
  ]
}
```

## 高级功能

### 1. 错误处理

每个步骤可以配置错误处理策略：

- **停止执行**：遇到错误立即停止整个工作流
- **继续执行**：记录错误但继续执行后续步骤

### 2. 超时控制

为长时间运行的步骤设置超时时间（秒）：
- 0：不限制
- >0：超过指定时间自动终止

### 3. 步骤启用/禁用

- 可以临时禁用某个步骤
- 被禁用的步骤会被跳过
- 用于调试或临时关闭功能

### 4. 条件执行（计划中）

- 根据前置步骤的结果决定是否执行
- 支持简单的条件表达式

## 最佳实践

### 1. 命名规范

- **工作流编码**：使用小写字母和下划线，如 `title_to_article`
- **步骤名称**：简洁描述步骤功能，如 "获取标题"
- **参数名称**：使用驼峰命名，如 `siteId`

### 2. 参数映射

- 优先使用变量引用而非常量
- 为常量值添加注释说明
- 检查参数类型匹配

### 3. 错误处理

- 关键步骤使用"停止执行"
- 可选步骤使用"继续执行"
- 记录详细的错误日志

### 4. 性能优化

- 合理设置超时时间
- 避免不必要的步骤
- 优化工具的执行效率

### 5. 测试验证

- 使用模拟执行测试工作流
- 验证各种边界情况
- 检查错误处理逻辑


## 故障排查

### 问题1：工具列表加载失败
- 检查原子工具是否已启用
- 确认原子工具的输入输出定义正确

### 问题2：参数映射验证失败
- 检查前置步骤是否存在
- 确认字段路径拼写正确
- 验证数据类型匹配

### 问题3：模拟执行失败
- 查看详细错误信息
- 检查每个步骤的参数配置
- 确认工具可正常执行

## 技术架构

## 常见问题

**Q: 如何复用已有的工作流？**
A: 可以使用【复制】功能创建副本，或将工作流保存为模板。

**Q: 参数映射支持哪些表达式？**
A: 当前仅支持简单的变量引用，复杂表达式功能计划中。

**Q: 如何调试工作流？**
A: 使用模拟执行功能，查看每个步骤的输入输出数据。

**Q: 工作流可以嵌套吗？**
A: 当前版本不支持，但可以将子工作流封装为原子工具。
