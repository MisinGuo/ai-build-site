# 游戏推广系统多语言支持解决方案

## 1. 现状分析

### 1.1 前端多语言支持现状 ✅
- **完善的前端国际化框架**：已实现基于 Next.js 的国际化路由和翻译系统
- **支持语言**：简体中文(zh-CN)、繁体中文(zh-TW)、英文(en)
- **路由机制**：
  - 默认语言(zh-CN)：无路由前缀，如 `/special/`
  - 其他语言：有路由前缀，如 `/zh-TW/special/`, `/en/special/`
- **翻译覆盖**：导航、操作、内容、表单等所有UI元素已完成翻译

### 1.2 后端多语言支持现状 ❌
目前只有**文章表(`gb_articles`)**支持多语言：
- 有 `locale` 字段(默认 'zh-CN')
- 有 `master_article_id` 字段支持文章间的语言关联

**缺乏多语言支持的核心数据表**：
- `gb_games` - 游戏表
- `gb_game_boxes` - 游戏盒子表  
- `gb_categories` - 分类表
- `gb_dramas` - 短剧表
- `gb_ai_platforms` - AI平台表
- 其他内容表

## 2. 解决方案设计

### 2.1 数据库架构方案

我们推荐采用**独立翻译表**方案，而非在主表中添加语言字段，原因：
1. 保持主表结构简洁
2. 支持动态添加新语言
3. 对现有代码影响最小
4. 便于管理翻译状态

### 2.2 核心设计原则

1. **主数据唯一性**：每个实体(游戏、盒子等)只有一条主记录
2. **翻译数据关联**：通过翻译表关联不同语言版本
3. **默认语言优先**：当某语言翻译不存在时，回退到默认语言(zh-CN)
4. **向后兼容**：现有代码无需大幅修改

## 3. 数据库实现方案

### 3.1 通用翻译表结构

```sql
-- 通用翻译表
CREATE TABLE `gb_translations` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `entity_type` varchar(50) NOT NULL COMMENT '实体类型：game, game_box, category, drama等',
  `entity_id` bigint NOT NULL COMMENT '实体ID',
  `locale` varchar(10) NOT NULL DEFAULT 'zh-CN' COMMENT '语言代码：zh-CN, zh-TW, en',
  `field_name` varchar(100) NOT NULL COMMENT '字段名称：name, description, subtitle等',
  `field_value` text COMMENT '字段值',
  `create_by` varchar(64) DEFAULT '' COMMENT '创建者',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_by` varchar(64) DEFAULT '' COMMENT '更新者',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_translation` (`entity_type`, `entity_id`, `locale`, `field_name`),
  KEY `idx_entity` (`entity_type`, `entity_id`),
  KEY `idx_locale` (`locale`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='通用翻译表';
```

    private String locale;

    @TableField("field_name")

### 4.2 翻译服务类

```java
@Service
public class TranslationService {
    
    @Autowired
    private GbTranslationMapper translationMapper;
    
    /**
     * 获取实体的多语言字段值
     */
    public String getTranslatedField(String entityType, Long entityId, 
                                   String fieldName, String locale) {
        LambdaQueryWrapper<GbTranslation> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(GbTranslation::getEntityType, entityType)
               .eq(GbTranslation::getEntityId, entityId)
               .eq(GbTranslation::getFieldName, fieldName)
               .eq(GbTranslation::getLocale, locale);
               
        GbTranslation translation = translationMapper.selectOne(wrapper);
        return translation != null ? translation.getFieldValue() : null;
    }
    
    /**
     * 保存翻译
     */
    public void saveTranslation(String entityType, Long entityId, 
                               String fieldName, String locale, String value) {
        GbTranslation translation = new GbTranslation();
        translation.setEntityType(entityType);
        translation.setEntityId(entityId);
        translation.setFieldName(fieldName);
        translation.setLocale(locale);
        translation.setFieldValue(value);
        
        // 使用 INSERT ... ON DUPLICATE KEY UPDATE 语法
        translationMapper.insertOrUpdate(translation);
    }
    
    /**
     * 批量获取实体翻译
     */
    public Map<String, String> getEntityTranslations(String entityType, Long entityId, String locale) {
        LambdaQueryWrapper<GbTranslation> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(GbTranslation::getEntityType, entityType)
               .eq(GbTranslation::getEntityId, entityId)
               .eq(GbTranslation::getLocale, locale);
               
        List<GbTranslation> translations = translationMapper.selectList(wrapper);
        return translations.stream()
                .collect(Collectors.toMap(
                    GbTranslation::getFieldName,
                    GbTranslation::getFieldValue
                ));
    }
}
```

### 4.3 游戏实体扩展

```java
@Data
public class GbGameWithTranslations extends GbGame {
    
    // 翻译后的字段
    private String translatedName;
    private String translatedDescription;
    private String translatedSubtitle;
    
    // 构造方法，应用翻译
    public static GbGameWithTranslations fromGame(GbGame game, 
                                                  Map<String, String> translations) {
        GbGameWithTranslations result = new GbGameWithTranslations();
        BeanUtils.copyProperties(game, result);
        
        // 应用翻译，如果翻译不存在则使用原值
        result.setTranslatedName(translations.getOrDefault("name", game.getName()));
        result.setTranslatedDescription(translations.getOrDefault("description", game.getDescription()));
        result.setTranslatedSubtitle(translations.getOrDefault("subtitle", game.getSubtitle()));
        
        return result;
    }
}
```

### 4.4 控制器层修改

```java
@RestController
@RequestMapping("/api/games")
public class GbGameController extends BaseController {
    
    @Autowired
    private IGbGameService gameService;
    
    @Autowired
    private TranslationService translationService;
    
    /**
     * 获取游戏列表（支持多语言）
     */
    @GetMapping("/list")
    public TableDataInfo list(GbGame gbGame, HttpServletRequest request) {
        // 从请求头或参数获取语言
        String locale = getLocaleFromRequest(request);
        
        startPage();
        List<GbGame> list = gameService.selectGbGameList(gbGame);
        
        // 应用翻译
        List<GbGameWithTranslations> translatedList = list.stream()
                .map(game -> {
                    Map<String, String> translations = translationService
                            .getEntityTranslations("game", game.getId(), locale);
                    return GbGameWithTranslations.fromGame(game, translations);
                })
                .collect(Collectors.toList());
        
        return getDataTable(translatedList);
    }
    
    private String getLocaleFromRequest(HttpServletRequest request) {
        // 优先从路径参数获取
        String locale = request.getHeader("Accept-Language");
        if (StringUtils.isBlank(locale)) {
            locale = "zh-CN"; // 默认语言
        }
        return locale;
    }
}
```

## 7. AI自动翻译集成

### 7.1 翻译服务

```java
@Service
public class AutoTranslationService {
    
    @Autowired
    private AIService aiService;
    
    @Autowired
    private TranslationService translationService;
    
    /**
     * 自动翻译实体字段
     */
    public void autoTranslateEntity(String entityType, Long entityId, 
                                   String fromLocale, String toLocale) {
        // 获取源语言的所有字段
        Map<String, String> sourceTranslations = translationService
                .getEntityTranslations(entityType, entityId, fromLocale);
        
        // 逐个字段进行翻译
        for (Map.Entry<String, String> entry : sourceTranslations.entrySet()) {
            String fieldName = entry.getKey();
            String sourceValue = entry.getValue();
            
            // 调用AI翻译
            String translatedValue = aiService.translate(sourceValue, fromLocale, toLocale);
            
            // 保存翻译结果
            translationService.saveTranslation(entityType, entityId, 
                                             fieldName, toLocale, translatedValue);
        }
    }
}
```

## 9. 注意事项

### 9.1 性能考虑
1. **缓存策略**：为翻译数据添加Redis缓存，减少数据库查询
2. **批量查询**：避免N+1查询问题，使用批量查询翻译数据
3. **延迟加载**：仅在需要时加载翻译数据

### 9.2 数据一致性
1. **主数据优先**：翻译不存在时始终回退到主表数据
2. **翻译版本控制**：记录翻译的创建和修改时间
3. **数据同步**：主数据修改时提示更新相关翻译

### 9.3 用户体验
1. **加载优化**：翻译加载失败时不影响页面基本功能
2. **编辑体验**：提供友好的翻译编辑界面
3. **预览功能**：支持实时预览不同语言的页面效果

## 10. 总结

这套多语言支持解决方案的优势：

1. **架构清晰**：通过独立翻译表实现，保持主表结构简洁
2. **扩展性强**：可以轻松支持新语言和新字段
3. **性能友好**：支持缓存和批量查询优化
4. **向后兼容**：对现有代码改动最小
5. **管理便捷**：提供完整的后台管理界面
6. **AI增强**：集成自动翻译，提高翻译效率

通过这套方案，游戏推广系统将具备完整的多语言支持能力，为不同地区的用户提供本地化的体验。
